## Commands

- `npm run serve` — local Eleventy dev server with live reload
- `npm run build` — static build to `dist/`

No tests, linter, or CI beyond Netlify's deploy.

## Architecture

Personal gif collection, statically generated by Eleventy (v0.11) and deployed to Netlify. Also a PWA (manifest + service worker).

Content flow:
- Raw `.gif` files live in `/g/` at repo root. Dropping a new gif in `/g/` is the entire "add a gif" workflow — no index, no registry.
- `src/_data/helpers.js` exposes `helpers.gifs()`, which globs `./g/*.gif` and returns filenames (sans extension). Every template consumes this list.
- `src/index.njk` renders the HTML grid (one `<img>` per gif).
- `src/gifs.njk` renders `/gifs.json` — the public JSON feed consumed by the companion [Alfred workflow](https://github.com/starzonmyarmz/gifz-alfred-workflow). Keep the shape (`keywords`, `url`) stable — external clients depend on it.
- `.eleventy.js` passes `/g/` through untouched and copies PWA assets (`manifest.json`, `sw.js`, icons) from `src/` to `dist/`.
- Filename slug = display text. `titlize` filter swaps hyphens for spaces, so name files kebab-case (e.g. `andy-hits-computer.gif`).

Layout: `src/_includes/default.njk` holds all CSS and the click-to-copy-URL script inline — there is no separate CSS/JS build step.
